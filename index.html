Arquitectura del Computador

La arquitectura del computador describe el diseño conceptual y la organización funcional de un sistema de cómputo. Define cómo los componentes de hardware interactúan para ejecutar programas.

Componentes principales

  • CPU
  • Memoria
  • Entrada / Salida
  • Buses

CPU (Unidad Central de Procesamiento)

La CPU es el cerebro del computador encargado de ejecutar instrucciones.

Funciones principales

  • Leer instrucciones desde memoria
  • Decodificar instrucciones
  • Ejecutar operaciones
  • Controlar flujo de datos
  • Tomar decisiones en programas

Componentes del procesador

  • Unidad de Control: coordina la ejecución de instrucciones.
  • ALU: realiza operaciones aritméticas y lógicas.
  • Registros: memoria rápida dentro del CPU.
  • Cache: memoria de alta velocidad.

Ciclo de Instrucción

  1. Fetch → buscar instrucción en memoria
  2. Decode → interpretar la instrucción
  3. Execute → realizar operación
  4. Memory → acceso a memoria
  5. Write Back → guardar resultado

Jerarquía de Memoria

Nivel Descripción
Registros Memoria más rápida dentro del CPU
Cache Memoria rápida intermedia
RAM Memoria principal
Disco / SSD Memoria secundaria

Buses

  • Bus de datos → transporta datos
  • Bus de direcciones → indica dirección de memoria
  • Bus de control → controla operaciones

Tipos de Arquitectura

Von Neumann

  • Datos e instrucciones comparten memoria

Harvard

  • Memoria de datos separada
  • Memoria de instrucciones separada

RISC

  • Conjunto reducido de instrucciones
  • Ejecución rápida

CISC

  • Conjunto complejo de instrucciones

Arquitectura MIPS

MIPS significa Microprocessor without Interlocked Pipeline Stages.

Características

  • Arquitectura RISC
  • 32 registros
  • Instrucciones de 32 bits
  • Arquitectura Load / Store

Pipeline de MIPS

Etapa Función
IFInstruction Fetch
IDInstruction Decode
EXExecute
MEMMemory Access
WBWrite Back

Manejo de Registros en MIPS

MIPS tiene 32 registros de propósito general.

RegistroUso
$zeroSiempre contiene 0
$v0-$v1Valores de retorno
$a0-$a3Argumentos
$t0-$t9Temporales
$s0-$s7Guardados
$spPuntero de pila
$raDirección de retorno

Lenguaje Ensamblador

Lenguaje de programación de bajo nivel que utiliza nemotécnicos para representar instrucciones de máquina.

Componentes

  • Instrucciones
  • Operadores
  • Etiquetas

Modos de Direccionamiento

  • Directo
  • Indirecto
  • Registro

Instrucciones comunes

Aritméticas

  • add
  • sub
  • mul
  • div

Lógicas

  • and
  • or
  • xor
  • not

Memoria

  • lw
  • sw

Interpretación de Instrucciones

Ejemplo:

add $t0, $t1, $t2

Interpretación:

$t0 = $t1 + $t2

Ejemplo:

lw $t0, 4($s0)

Interpretación:

$t0 = memoria[$s0 + 4]

Control de Flujo

  • j → salto
  • beq → branch if equal
  • bne → branch if not equal

Syscall

li $v0,1 move $a0,$t0 syscall
